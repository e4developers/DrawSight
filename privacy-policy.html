How DrawSight handles inspection data.

This policy describes how the current DrawSight mobile app handles reports, photos, notes, AI requests, backups, logs, and related information. The wording is written around the app's real current local-first setup.

Effective date: April 9, 2026 Applies to current mobile builds

1. Overview

DrawSight is currently a local-first inspection reporting app. In normal use, inspection records, findings, notes, photos, and local recovery data are stored on the inspector's device rather than in a DrawSight-run cloud account.

Current builds do not provide a first-party always-on hosted sync account platform. Some optional features do send information to third parties when the user chooses to use them, which is described below.

2. Information handled by the app

Information you enter or create

  • property addresses, inspection identifiers, and client-facing report content
  • inspection photos, annotations, finding notes, and review edits
  • inspector profile details such as company name, license number, and signature image
  • backup files and PDF reports created by the user

Locally stored technical data

  • workflow preferences and app settings stored on the device
  • local crash logs and local telemetry events kept on-device
  • the user's Anthropic API key, stored through secure device storage

3. How DrawSight stores data

  • inspection data is primarily stored on the user's device
  • current builds do not provide a DrawSight-run always-on sync account
  • DrawSight does not automatically upload inspection records to a DrawSight server in current builds

Because the app is local-first, the device owner typically controls most retention, deletion, export, and restore decisions directly.

4. Third-party services

Anthropic

When the user enables and uses AI features, relevant inspection photos, prompts, notes, and related request content may be sent directly from the app to Anthropic for processing. Current builds use the user's own Anthropic API key for these requests.

Anthropic privacy information is available at anthropic.com/privacy.

Address lookup provider

When address autocomplete is used, typed property-address queries may be sent to the app's configured geocoding provider. Current app builds are configured to use OpenStreetMap Nominatim for that lookup path.

System share destinations

When the user exports a report, backup, or log file, the content is sent only to the destination selected by the user through the device's share sheet or file picker.

5. How information is used

DrawSight uses handled information to:

  • create and edit inspection reports
  • generate optional AI-assisted draft text and photo classifications
  • export PDF reports and encrypted backups
  • support local recovery, diagnostics, and product-quality improvements

6. Backups, exports, and restore behavior

  • backup files can contain sensitive inspection data, including photos, notes, names, and addresses
  • PDF reports and files are exported only when the user chooses to export them
  • once a file is shared or saved outside the app, its retention and security depend on the user and chosen destination

7. Retention and deletion

  • inspection data remains on-device until edited, deleted, or removed with the app
  • backups and exports remain wherever the user saves or shares them
  • logs remain local unless the user explicitly shares them
  • current builds do not have a DrawSight cloud account system that requires an account-deletion flow

8. Your choices

You generally control DrawSight data by:

  • editing or deleting inspections inside the app
  • deleting exported reports or backup files you created
  • removing the app or clearing app data from your device
  • choosing whether to use AI-assisted features at all

If you are a homeowner, resident, or client seeking access to inspection information, you should generally contact the inspection business that collected the data.

9. Security

DrawSight uses reasonable measures intended to reduce risk, including:

  • local encrypted storage for certain sensitive app data
  • secure device storage for the Anthropic API key
  • encrypted backup export support
  • HTTPS requests for third-party network calls

No device, network, or software environment can guarantee absolute security. Users remain responsible for securing their devices, external storage locations, and third-party provider accounts.

10. Children's privacy

DrawSight is intended for adult inspectors and inspection businesses. It is not directed to children.

11. Policy changes

This Privacy Policy may be updated as DrawSight's features, providers, and release setup change. The latest version is reflected by the effective date at the top of this page.
