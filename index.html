Built around the actual inspection process

DrawSight is not trying to be a generic note app. It is built for inspectors who want to move quickly in the field, keep reports organized, and use AI as a speed multiplier rather than a replacement for judgment.

Capture

Start with the property, not the paperwork

The workflow begins with photos, findings, and on-site notes so inspectors can document the job while they are still inside the inspection.

Draft

Use AI where it saves real time

AI helps with draft language, grouping, and photo-heavy review paths, especially when there is a large batch of findings to sort through quickly.

Deliver

Keep the inspector in charge of the final report

DrawSight is built around review, editing, and approval before export so the final client-facing document still reflects the inspector's judgment.

What the workflow is aiming to feel like

  1. Set up the inspection fast and start capturing photos immediately.
  2. Let batch workflows and AI-assisted drafting do the repetitive work.
  3. Review, reroute, edit, and approve findings in a tighter loop before export.
  4. Generate reports and backups without depending on a heavy always-on cloud stack.

Where DrawSight is strongest right now

The product is most differentiated when it leans into field speed, photo-first capture, and turning inspection media into draft report structure faster.

Photo Review

Batch-heavy inspection flow

DrawSight is built to handle photo-heavy inspections where review speed and organization matter as much as polished final output.

AI Assist

Draft text, route faster

Optional AI tools support finding generation, classification, and other repetitive draft work while leaving the final decision with the inspector.

Recovery

Built with interrupted field work in mind

Export, backup, and recovery paths matter because inspections do not happen in perfect conditions. The app is designed around that reality.

Local Control

Closer to the device, closer to the job

Current builds keep inspection data primarily on-device and only send data out when the inspector enables AI or explicitly exports a file.

Current release setup

DrawSight is still in a practical early-stage product shape, so this site is intentionally honest about how the current build works.

  • inspection records are primarily stored on-device
  • AI requests currently go directly to Anthropic when enabled
  • current builds still use a user-supplied Anthropic API key
  • reports, backups, and files leave the app only when the user chooses to export them
